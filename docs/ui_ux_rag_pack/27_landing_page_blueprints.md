# 27 — Landing Page Blueprints

Purpose: help agents design strong marketing/product landing pages.

## Landing page structure

```txt
1. Header / nav
2. Hero
3. Social proof or credibility
4. Problem explanation
5. Product demo / visual
6. Features grouped by outcome
7. Workflow / how it works
8. Use cases / personas
9. Comparison / why different
10. Pricing or CTA
11. FAQ
12. Footer
```

## Hero rules

Hero must answer in 5 seconds:

- What is this?
- Who is it for?
- Why should I care?
- What can I do next?

Use:

```txt
Headline: outcome-focused
Subheadline: specific product explanation
Primary CTA: action verb
Secondary CTA: demo/docs/GitHub/etc.
Visual: real product UI, not fake abstract decoration
```

## Feature section rules

Bad:
- Random list of features.

Good:
- Group features by user outcome.

Example:

```txt
Outcome: Ship faster
Features: templates, automation, preview, deploy hooks

Outcome: Stay in control
Features: audit logs, rollback, permissions, local data
```

## Visual style recipes

### Developer product
```txt
Dark or neutral theme
Mono accents
Code snippets / terminal cards
Compact spacing
Clear docs/GitHub CTA
```

### AI product
```txt
Calm surfaces
Conversation/workflow preview
Trust/security section
Examples and limitations
Human control emphasized
```

### SaaS business product
```txt
Clean light theme
Outcome metrics
Customer proof
Dashboard screenshots
Simple pricing
```

## Landing page anti-patterns

- Giant vague AI headline.
- Feature grid with no story.
- Abstract gradients but no product screenshot.
- CTA hidden below fold.
- No proof, no examples, no pricing hint.
- Too much animation before explaining value.
