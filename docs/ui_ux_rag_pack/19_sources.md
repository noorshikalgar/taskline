# Researched Source Map

This pack is synthesized from established design-system and UX research sources. Use these as source anchors for deeper retrieval or future expansion.

## Design systems and platform guidelines

- Material Design 3: open-source design system with guidelines, styles, and components for user-friendly interfaces. Important for color roles, components, dynamic color, and accessibility.
- Apple Human Interface Guidelines: best practices for Apple-platform experience design, layout, controls, navigation, and accessibility.
- Microsoft Fluent 2: design principles, color palettes, accessibility, and motion guidance for Microsoft-style product experiences.
- IBM Carbon Design System: open-source design system for IBM products; useful for enterprise UI, color, data visualization, and AI UI patterns.
- Atlassian Design System: useful for productivity/product collaboration UI, empty states, components, and content guidance.
- GOV.UK Design System: strong evidence-based patterns for forms, validation, error summaries, and plain-language service design.
- Radix UI Primitives: accessible unstyled primitives useful for building custom modern design systems.
- WAI-ARIA Authoring Practices / W3C: interaction patterns for accessible menus, dialogs, comboboxes, tabs, and other complex widgets.

## UX research and heuristics

- Nielsen Norman Group: usability heuristics, visibility of system status, progress indicators, skeleton screens, empty states.
- Baymard Institute: ecommerce and form UX research; especially validation, checkout, error recovery, and form design.
- W3C WCAG 2.2: accessibility conformance principles and success criteria.

## Data visualization

- IBM Carbon data visualization guidance: chart selection, palettes, accessibility.
- IBM Design Language charts guidance: comparisons, trends, part-to-whole, correlations, relationships, maps.

## Additional references used for robust visual system layer

- Material Design 3 — design system, components, color, typography, elevation, shape: https://m3.material.io/
- Material Web theming — system tokens for color, typography, elevation, shape: https://material-web.dev/theming/material-theming/
- Apple Human Interface Guidelines — platform design and typography guidance: https://developer.apple.com/design/human-interface-guidelines
- Microsoft Fluent 2 Design Tokens — semantic token guidance for color, typography, spacing, elevation: https://fluent2.microsoft.design/design-tokens
- W3C WCAG 2.2 — accessibility requirements and success criteria: https://www.w3.org/TR/WCAG22/
- WAI WCAG 2.2 changes — added success criteria since WCAG 2.1: https://www.w3.org/WAI/standards-guidelines/wcag/new-in-22/
- IBM Carbon Color — neutral surfaces, primary action color, and purposeful status colors: https://carbondesignsystem.com/elements/color/overview/
- IBM Carbon Spacing — spacing tokens using multiples of 2, 4, and 8: https://carbondesignsystem.com/elements/spacing/overview/
