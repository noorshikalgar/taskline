# 30 — Mobile App Blueprints

Purpose: help agents design touch-friendly mobile product UI.

## Mobile priorities

- One primary task per screen.
- Large tap targets.
- Less text, clearer hierarchy.
- Bottom reach zones matter.
- Avoid desktop tables; use cards/lists.
- Use progressive disclosure.

## Touch target rules

```txt
Minimum target: around 44px
Comfortable target: 48px+
List row height: 52–64px
Primary bottom action: 48–56px height
Screen padding: 16px
```

## Navigation patterns

### Bottom tabs
Use for 3–5 main destinations.

### Top tabs
Use for sibling views inside a section.

### Drawer
Use for less frequent destinations.

### Bottom sheet
Use for contextual actions, filters, pickers, quick forms.

## Mobile form rules

- Use correct keyboard type.
- Keep labels visible.
- Break long forms into sections/steps.
- Use sticky submit only when helpful.
- Show validation near fields.
- Avoid tiny dropdowns; use sheets/pickers.

## Mobile list rules

- Use cards/rows, not wide tables.
- Put primary info first.
- Use secondary text sparingly.
- Swipe actions only as shortcuts, not only path.
- Provide search/filter in reachable places.

## Mobile state checklist

- Slow network loading.
- Pull-to-refresh if expected.
- Offline or reconnecting banner.
- Empty state with clear CTA.
- Permission prompts and denied state.
- Keyboard covering input.
- Long text wrapping.
- One-handed use.
