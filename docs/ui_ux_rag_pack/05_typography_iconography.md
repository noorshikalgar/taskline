# Typography and Iconography

Typography is interface infrastructure.

## Type scale

Use a small, consistent type scale:

- 12px: metadata, helper text, table secondary text
- 14px: default product body text
- 16px: comfortable body/input text
- 20px: section title
- 24-32px: page title
- 40px+: marketing/hero only

Line-height:

- Dense UI: 1.3-1.45
- Reading content: 1.5-1.7

## Labels

Good labels are specific and action-oriented.

Bad: `Submit`, `Click here`, `Manage`, `Data`
Good: `Create project`, `Export CSV`, `Invite member`, `Billing history`

## Icons

Icons are support, not language replacement.

Rules:

- Use text labels for primary and risky actions.
- Use icons alone only for universal actions like search, close, settings, copy, or when space is constrained and accessible labels exist.
- Keep one icon style: stroke width, corner radius, fill style.
- Do not mix emoji-like icons with enterprise system icons unless intentionally branded.

## Numbers and data

- Use tabular numbers in tables/dashboards.
- Keep units visible.
- Use consistent decimals.
- Avoid huge numbers without separators.
- Show time zone for scheduled events or logs.

## Content voice

Product copy should be:

- Clear
- Specific
- Calm
- Helpful
- Human but not overly cute

For errors, say what happened and how to fix it. For empty states, say what the user can do next.

## Edge cases

- Long labels in other languages.
- Names with mixed scripts.
- Code, paths, URLs, hashes: use monospace and copy buttons.
- Truncation: never hide the only differentiating part of a name.
