# Copywriting Rules

Small text changes make the UI feel more professional.

## Voice

DevThread should sound calm, precise, developer-friendly.

Use:

- `Write an update…`
- `No tasks selected for this release`
- `Select tasks below to include them in release notes.`
- `Search tasks, folders, and updates…`
- `Last updated Jun 12, 2026`
- `Local-first`

Avoid:

- Long explanations inside UI.
- Debug-like labels.
- Inconsistent casing.
- Too many exclamation marks.

## Casing

Use sentence case for UI labels:

- `Release notes`
- `Selected for this release`
- `Daily hours`
- `Logged days`

Use uppercase only for small eyebrow labels:

- `WORKLOG METRICS`
- `ACTIVE TASKS`
- `SELECTED FOR THIS RELEASE`

## Empty state examples

### Releases selected area

Title:

`No tasks selected for this release`

Description:

`Select tasks below to include their updates in the release notes.`

### Search empty before typing

Title:

`Search across DevThread`

Description:

`Find tasks, folders, releases, and timeline updates from one place.`

Examples:

`active tasks`, `blockers`, `release june`

### No search results

Title:

`No results found`

Description:

`Try a different task title, folder name, or update keyword.`

### Empty task timeline

Title:

`No updates yet`

Description:

`Add your first note, decision, blocker, or worklog entry.`

## Duration formatting

Use consistent duration strings:

- `2d 1h`
- `1d 7h`
- `4h 30m`
- `0m` only when truly zero.

Do not mix `1d 30m`, `1 day`, `12:08 AM` styles randomly. Keep human-friendly labels in cards and mono compact labels in status areas.
